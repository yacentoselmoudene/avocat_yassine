# تثبيت تطبيق مكتب المحاماة (APK) — Sideload Android

دليل التثبيت اليدوي لتطبيق `مكتب المحاماة` على هاتف Android، خارج Google
Play Store. الملف موجود تحت :

```
mobile/android/app/build/outputs/apk/release/app-release.apk
```

## نظرة سريعة
- حجم APK : ~2.9 MB
- Android 6.0 (Marshmallow / API 23) فما فوق
- لا يحتاج اتصالاً بالإنترنت إلا للوصول لخادم المكتب
- موقّع بشهادة `CN=Avocat Yassine`

---

## الطريقة 1 — adb (موصى بها للمطوّرين)

شرط مسبق : كابل USB + Android SDK platform-tools على الكمبيوتر.

```bash
# 1. فعّل "خيارات المطوّرين" على الهاتف :
#    إعدادات > حول الهاتف > اضغط على "رقم البنية" 7 مرات
#
# 2. فعّل "تصحيح USB" :
#    إعدادات > خيارات المطوّرين > تصحيح USB
#
# 3. وصّل الهاتف بكابل USB، اقبل مفتاح RSA على الهاتف.
#
# 4. تثبيت :
~/Library/Android/sdk/platform-tools/adb devices       # تحقق من ظهور الجهاز
~/Library/Android/sdk/platform-tools/adb install -r \
  mobile/android/app/build/outputs/apk/release/app-release.apk
```

العلامة `-r` تستبدل أي إصدار سابق دون فقدان بيانات المستخدم.

---

## الطريقة 2 — تنزيل من ملف (للمستخدم النهائي)

1. **انقل APK إلى الهاتف** بإحدى الوسائل :
   - عبر USB (نسخ إلى `Download/`)
   - عبر البريد الإلكتروني (ملحق ذاتي الإرسال)
   - عبر خدمة سحابية (Google Drive, Dropbox)
   - عبر QR code يربط برابط تحميل HTTPS

2. **اسمح بتثبيت التطبيقات من مصادر غير معروفة** :
   - **Android 8+** : عند فتح APK يطلب الهاتف الإذن للتطبيق الذي يفتحه
     (Chrome, Files, Drive…). اضغط "إعدادات" > فعّل "السماح من هذا
     المصدر".
   - **Android 7 وأقل** : إعدادات > الأمان > "مصادر غير معروفة" → فعّل.

3. **افتح APK** من مدير الملفات → اضغط "تثبيت".

## التشغيل الأول

عند فتح التطبيق ستظهر شاشة بسيطة تطلب **عنوان الخادم** :

```
https://office.example.ma
   أو
http://192.168.1.42:8003   (شبكة محلية داخل المكتب)
```

- إذا كان الخادم على نفس الشبكة الـ Wi-Fi : ضع IP المحلي + المنفذ.
- إذا كان الخادم منشوراً عبر HTTPS عمومي : ضع النطاق كاملاً.

العنوان يُحفظ تلقائياً ولن يُسأل عنه في المرّات اللاحقة. لتغييره : اضغط
"تغيير" في شاشة الترحيب.

## التحديث

أعد تنفيذ نفس خطوات التثبيت بـ APK جديد — Android سيتعرّف على نفس التوقيع
ويحفظ بيانات المستخدم.

> ⚠️ **هام** : إذا تغيّر مفتاح التوقيع (release.jks) بين إصداريْن، يطلب
> Android **إلغاء التثبيت قبل التحديث** وستُفقد جميع البيانات المحلية
> (cookies، URL محفوظ، الخ).

## استكشاف الأخطاء

| المشكلة | السبب الأرجح | الحل |
|---|---|---|
| "التطبيق لم يُثبَّت" | بقايا إصدار قديم بتوقيع مختلف | `adb uninstall ma.avocatyassine.maktab` ثم أعد التثبيت |
| شاشة بيضاء بعد إدخال URL | الخادم غير متاح أو شهادة TLS خاطئة | جرّب URL في متصفح الهاتف أولاً |
| "Mixed content" مرفوض | الخادم HTTP والتطبيق ينتظر HTTPS | الإعداد `cleartext=true` مفعّل، تحقق من سجل WebView عبر `chrome://inspect` |
| لا يحفظ الجلسة | ملفات تعريف الارتباط محظورة | حذف بيانات التطبيق ثم إعادة تشغيل |

## للنشر العمومي (Play Store)

عند الجاهزية للنشر على Google Play :
1. أنشئ keystore إنتاجي جديد (لا تستخدم `release.jks` الحالي للتطوير).
2. سجّل في Play Console واربط `applicationId = ma.avocatyassine.maktab`.
3. ارفع **AAB** (Android App Bundle) بدل APK :
   ```
   cd mobile/android
   ./gradlew bundleRelease
   ```
4. فعّل **Play App Signing** (يخزّن مفتاحك على خوادم Google).
